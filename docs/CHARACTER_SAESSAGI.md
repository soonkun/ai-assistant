# 캐릭터 가이드 — 새싹이(Saessagi)

이 프로젝트의 AI 비서 캐릭터는 **새싹이(Saessagi)**다. 새싹 돋은 하얀 둥근 몸체에 노란 발, 통통한 팔을 가진 마스코트.

## 파일 배치

```
assets/character/saessagi/
├── neutral.png      ← 기본 (제공됨)
├── happy.png        ← 제공됨
├── surprised.png    ← 제공됨
├── sad.png          ← 제공됨
├── worried_v2.png   ← 제공됨
├── thinking.png     ← 제공됨
├── sleepy.png       ← 제공됨
└── study.png        ← 제공됨 (시스템 상태 — 장기 작업 처리 중 표시)
```

모든 이미지는 **같은 해상도, 같은 중심점, 투명 배경 PNG**여야 한다. Builder는 이 파일들이 모두 있다고 가정하고 M_04 모듈을 구현한다. 제작은 비개발 작업이므로 병렬 진행 가능.

## 표정별 사양 (일러스트레이터용 지침)

공통: 몸체(흰색 둥근 실루엣), 잎사귀(머리 위 두 장), 뺨 홍조 둥근 원, 노란 발 두 개, 팔 두 개는 전부 유지. **눈·입·잎사귀 각도·뺨 색·부가 선**만 표정별로 다르다.

| 상태 | 눈 | 입 | 기타 | 용도 |
|---|---|---|---|---|
| neutral | 큰 원형 점 + 하이라이트 | `⌣` 살짝 웃는 입 | 기본 | LLM 발화 감정 (기본 상태) |
| happy | `^ ^` 곡선 | 더 크게 `∪` 벌린 입, 혀 끝 살짝 | 뺨 홍조 진하게 | LLM 발화 감정 |
| surprised | 큰 동그라미 (원본보다 30% 확대) | 작은 `o` | 잎사귀 살짝 위로 튐 | LLM 발화 감정 |
| sad | 아래로 처진 곡선 `\` `/` | 역 `∪` | 눈 아래 작은 물방울(선택) | LLM 발화 감정 |
| worried | 기본 눈 + 기울어진 눈썹 선 | 물결 `~` | 뺨 홍조 연하게 | LLM 발화 감정 |
| thinking | 옆 방향 (좌우 살짝 치우친 하이라이트) | 살짝 비스듬 `⌢` | 잎사귀 하나 쫑긋 | LLM 발화 감정 |
| sleepy | 수평선 `— —` | 작게 다문 입 | 잎사귀 살짝 처짐, `zZ` 아이콘 옆에(선택) | LLM 발화 감정 |
| study | 큰 원형 점 + 하이라이트, **둥근 검정 테 안경** 착용 | 작고 귀여운 `⌣` (neutral 계열) | **붉은 책을 두 팔로 펼쳐 듦**, 양쪽 잎사귀 바짝 세움, **우측 상단 반짝임 `✦ ✦`**, 뺨 홍조 연하게 | **장기 작업 처리 중 표시 (LLM 발화 감정 아님)** |

## 립싱크용 추가(옵션, 품질 업그레이드 시)

`mouth_*.png` 4~5종을 따로 만들면 TTS 음성에 맞춰 입만 바꿔치기 가능:
- `mouth_closed.png`
- `mouth_a.png` (벌린)
- `mouth_i.png` (옆으로)
- `mouth_o.png` (동그랗게)

입을 머리와 분리해서 투명 PNG로 만들고, 몸체 쪽에는 입 자리가 비어있는 버전이 필요함. 이건 M_04의 B안(레이어 분리) 때 쓴다.

## 아이들 애니메이션 (CSS/프로그래밍으로 처리)

정지 이미지라도 "살아있는 느낌"을 주기 위한 효과. 코드에서 CSS transform으로 처리 — 아트워크는 추가 필요 없다.

- **숨쉬기**: `transform: scaleY(1.0 → 1.02 → 1.0)` 를 3초 주기 반복.
- **깜빡임**: 5~10초마다 눈 부분을 0.15초간 살짝 닫힌 버전으로 바꿈. 이건 따로 `blink.png`가 있으면 더 자연스럽지만, 없으면 `opacity: 1 → 0.6 → 1` 로 대신.
- **흔들림**: 말할 때 전체가 0.5° 좌우로 흔들림 (`rotate(-0.5deg → 0.5deg)` 주기 0.4초).
- **반응 바운스**: 이벤트(알림 도착 등) 시 `translateY(0 → -8px → 0)` 한 번.

## 감정 태그 프로토콜

LLM이 응답 텍스트에 `[emotion:happy]` 식의 태그를 삽입한다. M_04는 이 태그를 파싱해 해당 PNG로 스왑.

예시:
```
[emotion:thinking] 잠시만요, 문서를 찾아볼게요...
[emotion:happy] 찾았어요! 예산 승인 절차는 15페이지에 있네요.
```

LLM 발화 중 **사용 가능 태그(7종)**: `neutral`, `happy`, `surprised`, `sad`, `worried`, `thinking`, `sleepy`.
모르는 태그가 오면 `neutral` 로 폴백.

**주의 — `study` 는 시스템 상태 감정**: `study` 는 백엔드(M_06 DocumentIngest 등)가 장기 작업 처리 중 시각 피드백으로 **직접 emit** 한다. LLM 응답 텍스트에서 `[emotion:study]` 를 사용하면 M_08 `extract_emotion` 이 **미지 키로 취급해 `neutral` 로 폴백**한다(`logger.warning` 1회 발생). 따라서 시스템 프롬프트는 `study` 를 LLM 사용 가능 태그로 **안내하지 말 것**. 이는 코드 수준의 방어(`_SPOKEN_EMOTIONS` 집합에서 의도적 제외)이며 프롬프트 드리프트에도 견고하다. 자세한 계약은 `specs/M_08_AvatarState_SPEC.md` §6.3 D-6 참조.

## M_04 모듈 설계 요점

- 렌더러 인터페이스 `AvatarRenderer` 추상 정의.
- 구현체 `SpriteAvatarRenderer` 로 V1 시작 (이 문서 기준).
- 향후 `Live2DAvatarRenderer` 를 같은 인터페이스로 추가 가능.
- 펫 모드(투명 창, 항상 위, 드래그 이동)는 렌더러와 독립된 창 관리 레이어.
- 상태 변화는 200~300ms 부드러운 페이드(crossfade)로 전환.

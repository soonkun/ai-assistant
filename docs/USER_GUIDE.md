## 1. 앱 화면 구성

새싹이 앱은 **캐릭터 위젯**과 **패널** 두 부분으로 구성됩니다.

```
┌───────────────────────────────────────────────────────┐
│  💬채팅  📅일정  📂문서  📋회의록  ⚙️설정      ✕  │  ← 탭
├───────────────────────────────────────────────────────┤
│                                                       │
│                  탭 내용 영역                         │
│                                                       │
└───────────────────────────────────────────────────────┘

          🌱  ← 새싹이 캐릭터 (클릭하면 패널 열기/닫기)
```

**기본 조작**

| 동작 | 결과 |
|---|---|
| 새싹이 클릭 | 패널 열기 / 닫기 |
| ✕ 버튼 클릭 | 패널만 숨김 (캐릭터는 화면에 유지) |
| 탭 전환 | 다른 탭으로 이동해도 회의록 작업 내용 보존 |

## 2. 대화하기

새싹이는 **마이크** 또는 **텍스트 입력**으로 대화할 수 있습니다.

### 음성 대화

마이크에 대고 말하면 자동으로 음성을 인식합니다.  
말이 끝나면 자동으로 감지해 응답을 시작합니다 (VAD 기반 — 별도 버튼 불필요).

### 텍스트 입력

채팅 탭 하단 입력창에 직접 타이핑 후 Enter를 누릅니다.

### 대화 예시

```
나:     이번 달 공동연구과제 진행 현황 어떻게 돼?
새싹이:  현재 등록된 문서를 기준으로 말씀드릴게요.
         3월 중간 보고서에 따르면 과제 달성률이 68%입니다.
         (출처: 2026_공동연구_중간보고서.pdf 4페이지)
```

```
나:     오늘 오후 일정 알려줘
새싹이:  오늘 오후 일정입니다.
         - 14:00  스마트팜 업체 미팅 (회의실 A)
         - 16:00  주간업무회의
```

## 3. 일정 관리

자연어로 일정을 추가하고 조회합니다. 새싹이는 **세 가지 방식**으로 일정을 알려드립니다.

---

### 3-1. 아침 브리핑 (매일 09:00)

앱이 실행 중이면 매일 오전 9시에 오늘의 일정을 자동으로 알려드립니다.

> *"좋은 아침이에요! 오늘 일정을 알려드릴게요.  
> 오전 10시에 공동연구과제 착수회의가 있고,  
> 오후 2시에 스마트팜 업체 미팅이 예정되어 있습니다."*

---

### 3-2. 10분 전 음성 알림

등록된 일정 **10분 전**에 새싹이가 음성으로 알려드립니다.

> *"10분 후 오후 2시에 스마트팜 업체 미팅이 있습니다. 준비하세요!"*

---

### 3-3. 2시간 연속 작업 휴식 권고

마우스·키보드를 쉬지 않고 **2시간 이상** 사용하면 새싹이가 휴식을 권합니다.

> *"2시간 동안 계속 일하셨어요. 잠깐 쉬시는 건 어떨까요?  
> 눈도 쉬게 해주시고, 물 한 잔 마시고 오세요!"*

> **참고**: 같은 알림은 30분 안에 반복되지 않습니다.

---

### 일정 추가

```
나:     내일 오전 10시에 공동연구과제 착수회의 2시간 잡아줘
새싹이:  내일(4월 27일) 오전 10:00 ~ 12:00
         공동연구과제 착수회의를 등록했어요!
```

```
나:     5월 8일 오후 2시에 스마트팜 업체 미팅 1시간 추가해줘
새싹이:  5월 8일(금) 오후 2:00 ~ 3:00
         스마트팜 업체 미팅을 등록했습니다.
```

```
나:     매주 월요일 9시 30분에 주간업무회의 1시간으로 잡아줘
새싹이:  매주 월요일 오전 9:30 ~ 10:30 주간업무회의를 등록했어요!
```

### 일정 조회

```
나:     이번 주 일정 알려줘
새싹이:  이번 주 일정입니다.
         - 4월 28일(월) 09:30  주간업무회의
         - 4월 29일(화) 14:00  농업인 교육 프로그램 검토
         - 4월 30일(수) 10:00  국립농업과학원 협력 회의
```

```
나:     오늘 뭐 있어?
새싹이:  오늘 일정은 오전 10시에 공동연구과제 착수회의 하나예요!
```

### 일정 삭제

```
나:     5월 8일 업체 미팅 취소해줘
새싹이:  5월 8일 스마트팜 업체 미팅을 삭제했어요.
```

## 4. 사내 문서 검색 (RAG)

미리 등록해둔 사내 문서에서 필요한 정보를 찾아드립니다.  
**문서 탭**에서 폴더를 만들고 파일을 등록합니다.

---

### 4-1. 지원 파일 형식

| 형식 | 확장자 |
|---|---|
| PDF | `.pdf` |
| 워드 | `.docx` |
| 파워포인트 | `.pptx` |
| 한글 | `.hwpx` |
| 텍스트 | `.txt`, `.md` |

---

### 4-2. 폴더 만들기

문서를 등록하기 전에 **폴더를 먼저 만드세요**. 주제나 업무별로 폴더를 구분하면 문서를 체계적으로 관리할 수 있습니다.

**문서 탭 → "폴더 추가" 버튼 클릭 → 폴더 이름 입력 → Enter**

예시 폴더 구성:

```
📁 사업 계획
📁 연구 보고서
📁 규정·지침
📁 회의 자료
📁 교육 프로그램
```

폴더 이름은 나중에 변경할 수 있습니다 (폴더명 더블클릭 → 수정 → Enter).

---

### 4-3. 문서 등록 방법

**방법 1: 파일 선택 버튼**
1. 문서 탭에서 업로드할 폴더를 선택합니다.
2. "파일 선택" 버튼을 클릭합니다.
3. 등록할 파일을 선택합니다 (여러 파일 동시 선택 가능).
4. 업로드가 시작되고 파일별 진행률이 표시됩니다.

**방법 2: 드래그 앤 드롭**
1. 탐색기(또는 Finder)에서 파일을 선택합니다.
2. 새싹이 문서 탭 위로 끌어다 놓습니다.

> **참고**: 파일 하나당 최대 **50MB**까지 업로드할 수 있습니다.

**업로드 중 동작**:
- 파일별 진행률 바가 표시됩니다.
- 새싹이가 `study` 표정으로 바뀝니다 (인덱싱 처리 중 표시).
- 인덱싱 완료 후 `neutral` 표정으로 돌아옵니다.

---

### 4-4. 인덱싱 원리

등록된 문서는 자동으로 분석되어 검색 가능한 상태가 됩니다.

1. **문서 파싱**: 파일 형식에 맞게 텍스트 추출
2. **청크 분할**: 500자 단위로 나누기 (50자 겹침)
3. **벡터 임베딩**: BGE-M3 AI 모델로 의미 벡터화
4. **저장**: 로컬 벡터 DB(LanceDB)에 저장

> 인덱싱은 파일 업로드 직후 자동으로 실행됩니다. 크기에 따라 수 초~수십 초 소요됩니다.  
> 같은 파일을 다시 업로드하면 기존 인덱스를 덮어씁니다 (중복 없음).

---

### 4-5. 문서 검색 사용 예시

```
나:     공동연구 중간보고서에서 달성률 확인해줘
새싹이:  문서에서 찾았어요. 2026년 상반기 과제 달성률은 68%이며,
         목표 대비 2%p 미달 상태입니다.
         (출처: 2026_공동연구_중간보고서.pdf 4페이지)
```

```
나:     출장비 신청 절차 알려줘
새싹이:  규정 문서에서 찾았습니다. 출장비는 출장 후 5일 이내에
         지출결의서를 작성해 총무팀에 제출하면 됩니다.
         (출처: 출장비지급규정.hwpx 7페이지)
```

```
나:     스마트팜 기술 관련 자료 있어?
새싹이:  관련 문서를 3건 찾았어요.
         1. 2026_스마트팜_보급사업_계획서.pdf (12페이지)
         2. ICT농업기술_도입가이드.docx (5페이지)
         3. 스마트팜_현장평가_결과.pdf (8페이지)
```

---

### 4-6. 문서 삭제

문서 탭 → 해당 문서 옆 삭제 버튼(🗑️)을 클릭하면 문서와 인덱스가 모두 삭제됩니다.  
폴더 삭제 시 폴더 내 문서도 함께 삭제할지 선택할 수 있습니다.

## 5. 회의록 자동 작성

**회의록 탭**에서 3단계로 회의 결과보고서(.hwpx)를 자동 작성합니다.

```
[Step 1 전사] → [Step 2 회의록 작성] → [Step 3 결과보고서 생성]
```

> **작업 내용 보존**: 탭을 전환하거나 패널을 닫아도 진행 내용이 사라지지 않습니다.

---

### Step 1 — 전사 (녹음 파일 → 텍스트)

**지원 오디오 형식**: WAV, MP3, M4A, OGG, FLAC, WEBM

**사용 방법**:

1. "오디오 파일 선택" 버튼을 눌러 회의 녹음 파일을 선택합니다.
2. 파일명이 표시되면 "전사 시작" 버튼을 클릭합니다.
3. AI(Whisper)가 음성을 텍스트로 변환합니다.
   - 처리 중에는 진행 상황이 실시간으로 표시됩니다.
   - 45분 회의 기준 약 3~10분 소요됩니다.
4. 완료되면 새싹이가 음성으로 알려줍니다:
   > *"전사가 완료되었어요. 확인해 보시고 회의록 작성을 시작해주세요."*
5. 전사 결과를 검토합니다. 잘못 인식된 부분이 있으면 직접 수정할 수 있습니다.
6. "전사본 저장(.txt)" 버튼으로 전사 텍스트를 저장할 수 있습니다.

---

### Step 2 — 회의록 작성 (텍스트 → 개조식 회의록)

1. Step 1의 전사 결과가 자동으로 입력됩니다.
   - 필요하면 직접 수정하거나, 이미 갖고 있는 녹취 텍스트를 붙여넣을 수도 있습니다.
2. **분량 선택** — 결과보고서의 상세도를 결정합니다:

   | 분량 | 내용 |
   |---|---|
   | **1페이지** | 핵심 결정 사항과 주요 논의 위주로 간결하게 |
   | **2페이지** | 논의 흐름, 쟁점, 근거, 세부 결정 사항까지 상세히 |

   > **중요**: 분량은 여기서 결정됩니다. 2페이지짜리 결과보고서를 원하면 **반드시 2페이지를 선택**하세요.  
   > 회의록 자체가 1페이지 분량으로 작성되면, 결과보고서도 1페이지 수준으로 나옵니다.

3. "회의록 작성" 버튼을 클릭합니다.
4. LLM이 개조식 회의록을 작성합니다 (약 30초~2분 소요).
5. 완료되면 새싹이가 음성으로 알려줍니다:
   > *"회의록 작성이 완료되었습니다. 확인해보시고 결과보고서 작성을 시작하세요."*
6. 내용을 검토·수정하고, "회의록 저장(.txt)" 버튼으로 저장할 수 있습니다.

---

### Step 3 — 결과보고서 생성 (.hwpx)

1. Step 2의 회의록이 자동으로 입력됩니다 (직접 수정도 가능합니다).
2. "결과보고서 생성" 버튼을 클릭합니다.
3. 한글(.hwpx) 문서를 자동 생성합니다 (약 10~30초 소요).
4. 완료되면 새싹이가 음성으로 알려줍니다:
   > *"결과보고서를 완성하였습니다. 내용을 검토하세요."*
5. "결과보고서 다운로드(.hwpx)" 버튼을 클릭하면 저장 위치를 선택하는 대화상자가 열립니다.

> **주의**: 생성된 파일은 **24시간 후 자동 삭제**됩니다. 필요한 파일은 반드시 저장해두세요.

---

### 결과보고서 구성

| 항목 | 내용 |
|---|---|
| 회의 제목 | LLM이 회의 내용에서 자동 추출 |
| 일시·장소 | 녹취록에 언급된 경우 추출, 없으면 오늘 날짜 자동 입력 |
| 참석자 | 녹취록 내 언급된 인물 자동 추출 |
| 주요 내용 | 개조식 (○ 항목 / - 세부 / * 부연) |
| 결정 사항 | 회의에서 확정된 사항 |
| 향후 계획 | 후속 조치, 담당자, 기한 |

---

### 결과보고서 예시 (1페이지)

```
○ 2026년 스마트팜 보급 확대 추진 계획 논의
  - 총 예산 50억 원, 전국 20개 시군 대상 시범 운영
    * 4월 말 공모 공고, 5월 중 대상지 선정 예정
○ 농업인 교육 프로그램 개편 방향 검토
  - 온라인 교육 비중 30%에서 50%로 상향 결정
    * 하반기 시범 운영 후 전면 시행
```

## 6. 펫 모드

**설정 탭**에서 "펫 모드"를 켜면 새싹이가 바탕화면 위에 항상 떠 있는 작은 위젯으로 바뀝니다.  
업무 화면을 방해하지 않고 항상 옆에 두고 사용할 수 있습니다.

---

### 펫 모드 특징

| 기능 | 설명 |
|---|---|
| **항상 위** | 어떤 앱 창이 열려도 새싹이가 가장 앞에 표시됩니다 |
| **클릭 관통** | 새싹이 뒤에 있는 앱을 그대로 클릭할 수 있습니다 |
| **드래그 이동** | 새싹이를 클릭·드래그해 원하는 위치로 옮깁니다 |
| **위치 저장** | 앱을 재시작해도 마지막 위치를 기억합니다 |

---

### 펫 모드 조작

**크기 조절**: 캐릭터 위에 마우스를 올리면 우측 하단에 흰 원형 핸들이 나타납니다. 드래그로 크기를 조절합니다 (60px~300px).

**종료**: 캐릭터 위에 마우스를 올리면 우측 상단에 빨간 × 버튼이 나타납니다. 클릭하면 앱이 종료됩니다.

**패널 열기**: 펫 모드에서도 새싹이를 클릭하면 채팅 패널이 열립니다.

**펫 모드 해제**: 설정 탭 → 펫 모드 끄기.

## 7. 설정

**설정 탭**에서 새싹이의 동작 방식을 변경할 수 있습니다.

---

### LLM 모델 선택

여러 AI 모델 중 원하는 것을 선택할 수 있습니다.  
드롭다운에서 모델을 고른 뒤 "모델 전환" 버튼을 클릭합니다.

---

### 음성 엔진

새싹이가 말하는 목소리를 선택합니다.

| 엔진 | 특징 |
|---|---|
| **시스템 TTS** | macOS 시스템 음성 사용. 자연스러운 목소리 (유나 등). |
| **MeloTTS** | 완전 오프라인 전용 한국어 음성. 단일 목소리. |

> **시스템 TTS** 선택 시 목소리 종류를 추가로 선택할 수 있습니다.

---

### 재생 속도

슬라이더로 말하는 속도를 조절합니다 (0.5배속 ~ 1.8배속, 기본 1.2배속).  
"목소리 테스트" 버튼으로 현재 설정을 바로 확인할 수 있습니다.

---

### 펫 모드 켜기/끄기

설정 탭에서 펫 모드를 토글합니다.

---

### WebSocket 주소

새싹이 백엔드 서버 주소를 변경합니다 (기본값: `ws://127.0.0.1:12393/client-ws`).  
일반적으로 변경할 필요가 없습니다.

## 9. 알아두세요

- 새싹이는 **완전 오프라인**으로 동작합니다. 인터넷 연결 없이 사용 가능하며, 모든 데이터는 사내 PC에만 저장됩니다.
- 음성 인식은 **한국어**에 최적화되어 있습니다.
- 생성된 회의록·결과보고서 파일은 **24시간 후 자동 삭제**됩니다. 필요한 파일은 미리 저장해두세요.
- **45분 이상 대화가 없으면** 새싹이가 졸린 표정(`sleepy`)으로 바뀝니다.
- **2시간 연속 작업** 시 휴식을 권하는 안내음성이 나옵니다 (같은 알림은 30분 이내 반복 없음).
- 문서 탭에 등록한 파일은 백엔드 서버가 실행 중인 동안만 검색 가능합니다. 서버 재시작 후에는 인덱스가 유지됩니다.

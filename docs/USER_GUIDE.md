## 1. 앱 화면 구성 (창 모드 · 펫 모드)

새싹이는 **두 가지 모드**로 쓸 수 있습니다. 설정에서 언제든 전환합니다.

### 창 모드 (데스크톱)
전체 화면을 차지하는 작업용 레이아웃입니다. 왼쪽 사이드바에서 탭을 고르고, 오른쪽에서 내용을 봅니다.

```
┌──────────────┬────────────────────────────────────┐
│ 🌱 새싹이     │                                    │
│ 📅 일정표     │                                    │
│ 📝 업무노트   │           탭 내용 영역             │
│ 📂 문서       │                                    │
│ 📋 회의록     │                                    │
│ ⚙️ 설정       │                                    │
└──────────────┴────────────────────────────────────┘
```

### 펫 모드
바탕화면 위에 작은 새싹이 캐릭터만 떠 있습니다. 캐릭터를 클릭하면 채팅/탭 패널이 팝업으로 열립니다. 업무 화면을 가리지 않고 곁에 두고 쓰는 모드입니다 (자세히는 7장).

```
          🌱  ← 새싹이 (클릭 → 패널 열기/닫기)
```

**기본 조작**

| 동작 | 결과 |
|---|---|
| 새싹이 클릭 | 패널 열기 / 닫기 |
| ✕ 버튼 | 패널만 숨김 (캐릭터는 유지) |
| 탭 전환 | 회의록·노트 등 진행 중 작업 내용 보존 |
| 모드 전환 | 설정에서 펫 ↔ 창 모드 전환 (펫 모드에선 캐릭터의 모드 버튼으로도 가능) |

## 2. 대화하기 (음성 · 텍스트 · 이미지)

새싹이는 **마이크 · 텍스트 · 이미지** 세 가지 방식으로 입력받습니다.

### 음성 대화
마이크에 대고 말하면 자동으로 인식합니다. 말이 끝나면 자동 감지(VAD)해 응답을 시작합니다 — 별도 버튼이 필요 없습니다.

### 텍스트 입력
채팅 입력창에 타이핑 후 Enter. (한국어 IME 조합 중 Enter는 무시되어 오발송이 없습니다.)

### 이미지 / 스크린샷 첨부
- 화면을 캡처해 **클립보드에서 바로 붙여넣기(Ctrl/⌘+V)** 하거나 파일로 첨부할 수 있습니다.
- 첨부한 이미지는 **비전(OCR) 전용 모델**이 글자를 그대로 읽어내고, 그 내용을 메인 모델이 받아 **이미지 속 텍스트·표·수치·날짜·담당자를 반영한 업무노트**로 정리해 줍니다 (5장 참고).
- 비전 전용 모델은 설정에서 지정합니다(8장). 메인 모델이 이미지를 못 읽는 경우에도 이 모델이 대신 판독하므로 한글 스크린샷도 인식됩니다.

### 답변 진행 상태 표시
새싹이가 답하는 동안 캐릭터 위에 점선 말풍선으로 현재 단계가 표시됩니다.

> *"생각하고 있어요…"* → *"문서를 찾고 있어요…"* → 답변

이 말풍선은 답변이 나오거나 대기 상태가 되면 자동으로 사라집니다.

### 대화 예시
```
나:     이번 달 공동연구과제 진행 현황 어떻게 돼?
새싹이:  현재 등록된 문서를 기준으로 말씀드릴게요.
         3월 중간 보고서에 따르면 과제 달성률이 68%입니다.
         (출처: 2026_공동연구_중간보고서.pdf 4페이지)
```

> **의도 자동 분류**: 입력을 받으면 새싹이가 먼저 "일정인지 / 문서 검색인지 / 내 업무이력인지 / 노트로 저장할지"를 짧게 판단한 뒤, 알맞은 기능으로 처리합니다. (설정에서 끄거나 분류용 모델을 따로 지정할 수 있습니다 — 8장.)

## 3. 일정 관리

자연어로 일정을 추가·조회·삭제합니다. 새싹이는 **세 가지 방식**으로 먼저 알려주기도 합니다.

### 3-1. 아침 브리핑 (매일 09:00)
앱이 켜져 있으면 매일 오전 9시에 오늘 일정을 음성으로 알려줍니다.
> *"좋은 아침이에요! 오늘은 오전 10시에 공동연구과제 착수회의가 있어요."*

### 3-2. 10분 전 음성 알림
등록된 일정 **10분 전**에 알려줍니다.
> *"10분 후 오후 2시에 스마트팜 업체 미팅이 있습니다. 준비하세요!"*

### 3-3. 2시간 연속 작업 휴식 권고
마우스·키보드를 **2시간 이상** 쉬지 않고 쓰면 휴식을 권합니다. (같은 알림은 30분 안에 반복되지 않습니다.)

### 일정 추가 / 조회 / 삭제
```
나:     내일 오전 10시에 공동연구과제 착수회의 2시간 잡아줘
새싹이:  내일(4월 27일) 오전 10:00 ~ 12:00 공동연구과제 착수회의를 등록했어요!

나:     매주 월요일 9시 30분에 주간업무회의 1시간으로 잡아줘
새싹이:  매주 월요일 오전 9:30 ~ 10:30 주간업무회의를 등록했어요!

나:     이번 주 일정 알려줘
새싹이:  - 4월 28일(월) 09:30  주간업무회의
         - 4월 29일(화) 14:00  농업인 교육 프로그램 검토

나:     5월 8일 업체 미팅 취소해줘
새싹이:  5월 8일 스마트팜 업체 미팅을 삭제했어요.
```

### 캘린더 탭(일정표)
채팅 외에 **일정표 탭**에서 월별 일정을 달력 형태로 확인할 수 있습니다.

## 4. 사내 문서 검색 (RAG)

미리 등록해둔 문서에서 근거를 찾아 **출처와 함께** 답합니다. **문서 탭**에서 폴더를 만들고 파일을 등록합니다.

### 4-1. 지원 파일 형식
| 형식 | 확장자 |
|---|---|
| PDF | `.pdf` |
| 워드 | `.docx` |
| 파워포인트 | `.pptx` |
| 한글 | `.hwpx` |
| 텍스트 | `.txt`, `.md` |

> 파일 하나당 최대 **50MB**.

### 4-2. 폴더 만들기
주제·업무별로 폴더를 먼저 만드세요. **문서 탭 → "폴더 추가" → 이름 입력 → Enter.** (폴더명 더블클릭으로 수정 가능.)

```
📁 사업 계획   📁 연구 보고서   📁 규정·지침   📁 회의 자료
```

### 4-3. 문서 등록 방법
- **파일 선택 버튼**: 폴더 선택 → "파일 선택" → 파일 선택(다중 가능) → 파일별 진행률 표시.
- **드래그 앤 드롭**: 탐색기/Finder에서 문서 탭 위로 끌어다 놓기.
- **채팅으로 첨부**: 채팅 입력창에 문서를 첨부하면 자동으로 "업무노트" 폴더로 분류되어 등록됩니다.

**등록 중 표시**: 새싹이가 **등록 전용 모습**으로 바뀝니다. 펫 모드에서 채팅으로 첨부하면 *"책장 포털에 책을 넣는 새싹이"* 영상이 캐릭터 자리에 재생되고, 등록이 끝나면 평소 모습으로 돌아옵니다. (이 영상 교체 방법은 `docs/RAG_업로드_캐릭터_영상.md` 참고.)

### 4-4. 인덱싱·검색 원리
등록된 문서는 자동 분석되어 검색 가능해집니다.

1. **문서 파싱**: 형식별 텍스트 추출 (문장 경계 기준 분할)
2. **청크 분할**: 약 500자 단위 / 50자 겹침
3. **벡터 임베딩**: BGE-M3 모델로 의미 벡터화
4. **저장**: 로컬 벡터 DB(LanceDB)에 저장 (문서가 많아지면 인덱스 자동 생성으로 검색 가속)

검색 시에는 **하이브리드 + 리랭킹**으로 정확도를 높입니다:
- **의미 검색(벡터)** 과 **키워드 검색(BM25)** 을 함께 돌려 RRF로 융합 → 과제번호·고유명사 같은 정확한 키워드도 잘 잡습니다.
- 융합 결과를 **리랭커(bge-reranker-v2-m3)** 가 질문과의 관련도로 재정렬해 상위만 답변 근거로 사용합니다.
- GPU가 있으면 FP16으로 가속됩니다(없어도 CPU로 동작).

> 같은 파일을 다시 올리면 기존 인덱스를 덮어씁니다(중복 없음). 서버를 재시작해도 인덱스는 유지됩니다.

### 4-5. 검색 사용 예시
```
나:     공동연구 중간보고서에서 달성률 확인해줘
새싹이:  2026년 상반기 과제 달성률은 68%로, 목표 대비 2%p 미달입니다.
         (출처: 2026_공동연구_중간보고서.pdf 4페이지)

나:     출장비 신청 절차 알려줘
새싹이:  출장 후 5일 이내 지출결의서를 작성해 총무팀에 제출하면 됩니다.
         (출처: 출장비지급규정.hwpx 7페이지)
```

### 4-6. 원본 바로 열기
답변에 붙는 **출처(참고자료) 칩**, 문서 탭의 파일, 업무노트의 **관련 자료**를 클릭하면 — 저장 위치를 묻지 않고 **기본 앱(한글·PDF 뷰어 등)으로 바로 열립니다.** (임시 사본을 열기 때문에 원본 인덱스는 그대로 보존되며, 보관이 필요하면 연 앱에서 "다른 이름으로 저장"하면 됩니다.)

### 4-7. 문서 삭제
문서 옆 삭제 버튼(🗑️)으로 문서와 인덱스를 함께 삭제합니다. 폴더 삭제 시 내부 문서 동시 삭제 여부를 선택할 수 있습니다.

## 5. 업무노트

회의·자료·화면을 **업무노트**로 정리해 모아둡니다. **업무노트 탭**에서 보고 편집합니다.

### 5-1. 노트 만드는 방법
- **채팅으로 자동 생성**: 자료(문서/이미지)와 함께 "오늘 회의 진행했어"처럼 한 줄만 적어도 됩니다. 새싹이가 **첨부 문서·이미지의 실제 내용을 근거로** 제목과 요약(상황·절차·사용 자료·교훈)이 들어간 업무노트를 자동으로 만듭니다. 노트 제목도 내용을 대표하도록 구체적으로 붙습니다.
- **스크린샷으로 생성**: 화면을 캡처해 채팅에 붙여넣으면, 비전 모델이 이미지 속 텍스트·표·수치를 읽어 업무노트로 정리합니다.
- **빈 노트 직접 작성**: 업무노트 탭의 **+ 버튼**으로 새 노트를 만들고 직접 씁니다.

> 노트를 작성·저장하는 동안 새싹이 아이콘이 **노트 작성 모드**로 바뀌고 "업무 노트를 작성할게요!"라고 안내합니다. 노트 안의 **관련 자료**를 클릭하면 원본 파일이 기본 앱으로 바로 열립니다.

### 5-2. 블록 에디터 (창 모드)
창(데스크톱) 모드에서는 **Notion 스타일 블록 에디터**로 노트를 작성합니다. 제목, 헤더, 목록, 코드 블록 등 기본 마크다운 서식을 지원하며 UI는 한국어입니다. (펫 모드에서는 가벼운 텍스트 편집기를 씁니다.)

### 5-3. 노트 그래프
업무노트 탭의 그래프 보기에서 **노트 간 연결 관계**를 네트워크 그래프로 볼 수 있습니다. 노트가 추가/수정되면 그래프가 자동으로 갱신됩니다.

### 5-4. 작성 중 표시
노트를 저장하는 동안 새싹이가 노트 작성 표정(`note_writing`)으로 바뀝니다.

> 업무노트는 마크다운 파일로 **내 PC(로컬)** 에 저장됩니다.

## 6. 회의록 자동 작성

**회의록 탭**에서 3단계로 회의 결과보고서(.hwpx)를 만듭니다. (탭을 옮기거나 패널을 닫아도 진행 내용은 보존됩니다.)

```
[Step 1 전사] → [Step 2 회의록 작성] → [Step 3 결과보고서 생성]
```

### Step 1 — 전사 (녹음 → 텍스트)
- 지원 오디오: WAV, MP3, M4A, OGG, FLAC, WEBM (오디오 파일 드래그앤드롭도 지원)
- "오디오 파일 선택" → "전사 시작" → Whisper가 텍스트로 변환 (45분 기준 약 3~10분)
- 완료 후 음성 안내. 잘못 인식된 부분은 직접 수정 가능, "전사본 저장(.txt)" 제공.

### Step 2 — 회의록 작성 (텍스트 → 개조식 회의록)
- Step 1 결과가 자동 입력됨(직접 붙여넣기도 가능).
- **분량 선택**이 핵심:

  | 분량 | 내용 |
  |---|---|
  | **1페이지** | 핵심 결정·주요 논의 위주로 간결 |
  | **2페이지** | 논의 흐름·쟁점·근거·세부 결정까지 상세 |

  > 2페이지 결과보고서를 원하면 **반드시 2페이지를 선택**하세요. 회의록이 짧으면 보고서도 짧아집니다.
- "회의록 작성" → LLM이 개조식 회의록 생성(약 30초~2분). "회의록 저장(.txt)" 제공.

### Step 3 — 결과보고서 생성 (.hwpx)
- Step 2 회의록이 자동 입력됨.
- "결과보고서 생성" → 한글(.hwpx) 자동 생성(약 10~30초) → "결과보고서 다운로드(.hwpx)"로 저장 위치 선택.

> 생성 파일은 **24시간 후 자동 삭제**됩니다. 필요한 파일은 꼭 저장하세요.

### 결과보고서 구성 / 예시
| 항목 | 내용 |
|---|---|
| 회의 제목·일시·장소·참석자 | 녹취록에서 자동 추출(없으면 오늘 날짜) |
| 주요 내용 | 개조식 (○ 항목 / - 세부 / * 부연) |
| 결정 사항 / 향후 계획 | 확정 사항, 후속 조치·담당·기한 |

```
○ 2026년 스마트팜 보급 확대 추진 계획 논의
  - 총 예산 50억 원, 전국 20개 시군 대상 시범 운영
    * 4월 말 공모 공고, 5월 중 대상지 선정 예정
○ 농업인 교육 프로그램 개편 방향 검토
  - 온라인 교육 비중 30%에서 50%로 상향 결정
```

## 8. 설정

**설정 탭**에서 동작 방식을 바꿉니다.

### 화면
- **테마**: 다크 🌙 / 라이트 ☀️
- **글씨 크기**: 펫 모드와 창 모드를 **각각 따로** 저장

### LLM (AI 모델)
- **공급자 선택**: **Ollama(로컬 모델)** 또는 **OpenAI(ChatGPT)**
- 모델 선택(드롭다운) 및 OpenAI API 키 입력
- "모델 전환"으로 적용

> 현재 OpenAI는 사내 인트라넷 GPU 인프라가 갖춰지기 전까지의 **임시 대역**입니다. 인트라넷 모델로 전환하면 외부 호출 없이 동작합니다.

### 비전(전사) 모델 — 이미지 OCR 전용
이미지/스크린샷의 글자를 읽는 일은 **메인 채팅 모델과 별도의 전용 모델**이 담당합니다. 설정 파일 `conf.yaml`의 `app.ollama.vision_model`에 지정합니다.

```yaml
app:
  ollama:
    model: gemma4:latest        # 메인 대화·도구 모델
    vision_model: qwen2.5vl:7b  # 이미지 OCR 전용 (비우면 라우팅 안 함)
```

- 한글 OCR이 강한 모델 권장(예: `qwen2.5vl:7b`). 다른 모델로 바꾸려면 이 값만 교체.
- 메인 모델(gemma4 등)이 이미지를 못 읽어도, 이미지가 첨부된 턴은 이 비전 모델이 글자를 전사한 뒤 메인 모델이 답변·노트를 작성합니다.
- 사용 전 `ollama pull <모델명>`으로 받아 두어야 합니다.

### 의도 분류기
입력을 자동 분류하는 기능을 켜고 끄거나, 분류 전용 모델(메인과 동일 / Ollama / OpenAI)을 지정합니다. (로컬 모델은 콜드스타트가 느릴 수 있어, 첨부가 있을 때는 분류가 늦어도 노트 작성으로 처리되는 안전망이 있습니다.)

### 지침 관리 (고급)
새싹이의 행동 지침(프롬프트)을 UI에서 직접 편집·저장할 수 있습니다. 페르소나, 업무노트 작성 가이드, 문서 답변 가이드, 내 업무이력 답변 가이드, 의도 분류, 회의록 작성 가이드 등이며, 각 항목에 위험도가 표시되고 기본값 복원 버튼이 있습니다.

### 음성 (TTS)
| 엔진 | 특징 |
|---|---|
| **시스템 TTS** | OS 시스템 음성(유나 등). 목소리 종류 추가 선택 가능 |
| **MeloTTS** | 완전 오프라인 한국어 음성(단일 목소리) |
- **재생 속도**: 0.5~1.8배(기본 1.2배). "목소리 테스트"로 즉시 확인.

### 펫 모드 / WebSocket 주소
- 펫 모드 토글.
- 백엔드 서버 주소(기본 `ws://127.0.0.1:12393/client-ws`) — 일반적으로 변경 불필요.

## 10. 활용 시나리오

여러 기능을 엮은 실제 업무 흐름 예시입니다.

### 시나리오 A — 회의 끝나고 결과보고서까지
1. 회의 녹음 파일을 **회의록 탭**에 넣고 전사 → 회의록(2페이지 선택) → 결과보고서(.hwpx) 생성·저장.
2. 회의에서 쓰인 자료 PDF를 **문서 탭**(또는 채팅 첨부)으로 등록.
3. 채팅에서 *"방금 올린 회의 자료에서 결정사항만 뽑아줘"* → 출처와 함께 답변.

### 시나리오 B — 규정·절차 빠른 확인
1. 규정·지침 문서들을 **문서 탭 "규정·지침" 폴더**에 등록.
2. 펫 모드로 곁에 두고, 작업 중 떠오를 때마다 *"출장비 신청 절차 알려줘"*, *"연차 이월 규정?"* 질문 → 근거 페이지까지 답변.

### 시나리오 C — 화면 캡처를 업무노트로
1. 참고할 화면(대시보드·표·메일)을 캡처해 채팅에 **붙여넣기(⌘+V)**.
2. *"이 화면 업무노트로 정리해줘"* → 비전 AI가 수치·표를 읽어 노트 자동 작성.
3. **업무노트 탭**에서 블록 에디터로 다듬고, **그래프 보기**로 관련 노트와의 연결을 확인.

### 시나리오 D — 아침 업무 루틴
1. 출근해 앱을 켜두면 09:00 **아침 브리핑**으로 오늘 일정 청취.
2. 새 일정이 생기면 *"오후 3시에 현장점검 1시간 잡아줘"* 로 바로 등록.
3. 일정 **10분 전 음성 알림**으로 놓치지 않기. 2시간 몰입하면 **휴식 권고**.

### 시나리오 E — 곁에 두고 일하기 (펫 모드)
- 펫 모드로 새싹이를 화면 구석에 두고, 다른 앱은 그대로 클릭(클릭 관통)하며 작업.
- 궁금한 게 생기면 캐릭터 클릭 → 바로 질문. 문서를 채팅에 끌어다 놓아 그 자리에서 등록.

## 11. 알아두세요

- 노트·문서·일정·회의록 등 **데이터는 모두 내 PC(로컬)에 저장**됩니다.
- LLM은 설정에서 **로컬(Ollama)** 또는 **외부(OpenAI)** 를 선택합니다. 외부 모델 선택 시에만 해당 서비스로 호출이 나갑니다(향후 사내 인트라넷 모델로 전환 예정).
- 음성 인식·합성은 **한국어**에 최적화되어 있습니다.
- 생성된 회의록·결과보고서 파일은 **24시간 후 자동 삭제** — 필요한 건 미리 저장하세요.
- 오래 대화가 없으면 졸린 표정(`sleepy`)으로 바뀝니다.
- **2시간 연속 작업** 시 휴식 안내(같은 알림은 30분 내 반복 없음).
- 문서·노트 검색은 백엔드 서버가 켜져 있어야 동작합니다. 서버 재시작 후에도 인덱스는 유지됩니다.

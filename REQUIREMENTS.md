# REQUIREMENTS — 사내 인트라넷 AI 비서

## 0. 배포 환경

- **완전 오프라인**: 사내 인트라넷 PC, 외부 인터넷 차단.
- **OS (배포 타깃)**: Windows 10/11 **전용**. 배포 번들·인스톨러는 Windows 10/11 전용 유지.
  - **개발 환경**: macOS 14+ (Apple Silicon), Linux (x86-64/ARM) 지원.
    배포 번들·인스톨러는 Windows 10/11 전용 유지.
  - **GPU 가속**: Windows: CUDA (NVIDIA), macOS: MPS (Metal), Linux: CUDA 또는 CPU 자동 감지.
- **하드웨어 전제**: CPU 8코어 이상, RAM 16GB 이상, GPU 없어도 동작해야 함(있으면 가속).
- **설치 방식**: 오프라인 번들 (모델 GGUF, Python 휠, npm 패키지 캐시 포함)로 단일 인스톨러 제공.

## 1. 대화 기능

### 1.1 음성 대화
- STT(Speech-to-Text): 한국어, 영어 지원. 마이크 입력 연속 청취.
- VAD(Voice Activity Detection)로 발화 구간만 전송.
- TTS(Text-to-Speech): 한국어 여성 목소리 기본. 음성 클로닝(Voice Clone) 옵션.
- **전이중(Full Duplex)**: 사용자가 AI 발화 중간에 끼어들면 AI는 즉시 멈추고 듣는다.

### 1.2 텍스트 대화
- 채팅 UI로 타이핑 입력도 받음.
- 음성 대화 히스토리와 텍스트 대화 히스토리는 동일 컨텍스트.

## 2. 문서 RAG

### 2.1 문서 등록
- 지원 포맷: PDF, DOCX, PPTX, HWP/HWPX, TXT, MD.
- 파싱 시 **페이지 번호·섹션·바운딩 박스를 메타데이터로 보존**.
- 임베딩은 페이지 단위가 아니라 "페이지 내 의미 단위" 청크.
- 벡터 DB는 로컬 디스크(Qdrant embedded 또는 LanceDB).

### 2.2 질의응답
- 질문에 대해 답변과 함께 **"이 내용은 `문서명.pdf` 12페이지, '예산 승인 절차' 섹션에 있습니다"** 형태의 인용을 제공.
- 인용 클릭 시 원문 PDF의 해당 페이지를 열어 하이라이트 표시.
- 관련 문서가 없으면 "등록된 문서에서 답을 찾지 못했습니다"라고 솔직히 답한다. 추측 금지.

## 3. 캐릭터 아바타

### 3.1 캐릭터
- 이 프로젝트의 공식 캐릭터는 **새싹이(Saessagi)** — `assets/character/saessagi/` 참조.
- 상세 사양: `docs/CHARACTER_SAESSAGI.md`.

### 3.2 렌더링 방식
- V1: **스프라이트 스왑 방식** (표정별 PNG 7종).
- 렌더러는 `AvatarRenderer` 인터페이스로 추상화해 향후 레이어 합성·Live2D 교체 가능.
- 펫 모드: 투명 배경, 항상 위, 마우스 클릭 관통.
- 드래그로 화면 아무 위치로 이동.

### 3.3 표현
- LLM 출력의 `[emotion:happy]` 같은 태그로 표정 전환.
- 표정 전환 시 200~300ms 페이드 crossfade.
- 아이들 애니메이션: 숨쉬기(scaleY 1.0→1.02), 깜빡임(5~10초 간격), 말할 때 좌우 흔들림.
- 립싱크: V1에선 간단한 opacity 펄스로 시작. 이후 입 레이어 분리 업그레이드.
- 감정 세트: 중립, 기쁨, 놀람, 슬픔, 걱정, 생각 중, 졸림 (7종).

## 4. 스케줄링·다이어리

### 4.1 일정 등록
- 음성 또는 텍스트로 "내일 오후 3시에 회의 있어" → 자연어 파싱으로 `{title, start, duration}` 구조화.
- Gemma의 함수 호출(function calling)을 사용해 `add_event()` 호출.
- SQLite로 로컬 저장.

### 4.2 알림
- 일정 도래 **10분 전 팝업 + 음성 알림**.
- 아침 첫 실행 시: **오늘의 일정을 캐릭터가 브리핑**.
- 특정 일정 조회: "내일 뭐 있어?" → 해당 날짜 이벤트 나열.

## 5. 휴식 권고 (유휴 감지)

- 마우스·키보드 입력이 **임계 시간**(기본 45분, 설정 가능) 없으면 가볍게 휴식 권고.
- 연속 업무 **임계 시간**(기본 2시간) 넘으면 "이제 그만 일하라"는 톤의 메시지.
- 메시지는 반복되지 않게 쿨다운 로직 적용.
- 사용자가 "방해 금지" 모드 설정 가능.

## 6. 화면 인식

- 사용자 트리거("화면 봐줘" 등)에 화면을 스크린샷.
- Gemma 4 E4B의 비전 입력으로 전달.
- 질문 맥락에 따라 응답: 코드 오류 설명, 문서 요약, UI 조작 안내 등.
- 연속 화면 공유 모드 있음(N초마다 자동 캡처, 개인정보 주의 경고).

## 7. MCP 확장 포인트

- DuckDuckGo 검색(인트라넷에선 비활성, 외부망 가능 PC에서만).
- 사내 위키/Confluence 검색 MCP(가능하면).
- 파일 시스템 MCP로 허용된 폴더만 읽기·쓰기.

## 8. 모델

- **LLM**: Gemma 4 E4B (Ollama `gemma4:e4b`) — 멀티모달, 함수 호출, 131K 컨텍스트.
- **임베딩**: BGE-M3 (한국어 포함 다국어).
- **STT**: faster-whisper large-v3 (또는 sherpa-onnx로 경량화).
- **TTS**: MeloTTS(한국어) 기본 + XTTS v2 옵션(음성 클로닝). ※ Piper 공식 한국어 모델 미존재로 교체.
- **리랭커**: Qwen3-Reranker-8B (필요 시에만).

## 9. 비기능 요구사항

- **기동 시간**: 15초 이내에 초기 화면 표시.
- **음성 응답 지연**: 사용자 발화 끝 → AI 첫 음성까지 목표 (스트리밍 TTS 첫 청크 기준).
  - GPU(NVIDIA 8GB+): **2초 이내**.
  - CPU-only: **6초 이내**.
- **메모리**: 전체 프로세스 RSS (Electron 포함, 모델 포함):
  - MIN 프로파일(RAM 16GB): **14GB 이하** — Whisper medium int8 사용.
  - RECOMMENDED 프로파일(RAM 32GB): **20GB 이하** — Whisper large-v3, 리랭커 optional.
- **프라이버시**: 어떤 형태로도 외부 네트워크 호출 금지. 위반 시 빌드 실패.
- **로그**: PII(개인정보)는 마스킹. 로컬 로그 보관 7일.

## 10. 제외사항 (Non-Goals)

- 학습/파인튜닝 기능은 V1에 포함 안 함.
- 다중 사용자 동시 접속 안 함(1인 1PC 모델).
- 모바일 앱 안 만듦.
- 음악 생성, 이미지 생성은 범위 밖.
